好的，我来分析一下。原查询的主要问题是缺少索引和使用了低效的 SELECT *。

这是优化后的版本：

sql 
SELECT
  o.id,
  o.customer_name,
  o.total_amount,
  o.status
FROM orders o
WHERE o.created_at >= '2024-01-01'
  AND o.status IN ('pending', 'processing')
ORDER BY o.created_at DESC
LIMIT 100;

主要优化点：

  • 指定字段：用具体列名替代 SELECT *，减少数据传输量
  • 索引建议：在 created_at 和 status 上创建复合索引
  • 分页限制：添加 LIMIT 防止全表扫描
